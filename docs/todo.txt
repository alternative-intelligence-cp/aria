Macro-Driven Development Pattern:
- Use %macro GEN_FUNCNAME for type-generic implementations
- Generate variants for all relevant types (int8-64, uint8-64, flt32-64)
- Functions return result types with autowrap (*) for safety
- Users unwrap with ? operator
- Maintain explicit type selection (no overloading)

Type Coverage per Function:
- Numeric operations: int8, int16, int32, int64, uint8, uint16, uint32, uint64, flt32, flt64
- Character operations: uint8 (ASCII/UTF-8 code points)
- String operations: string type
- Collections: Generic over element type

Testing Requirements:
- Each stdlib module needs comprehensive test file
- Test all type variants
- Verify edge cases (overflow, underflow, null, empty)
- All tests must pass with exit code 0

Documentation Standards:
- Function signature with parameter types
- Description of behavior
- Return value description
- Usage examples
- Edge case behavior
